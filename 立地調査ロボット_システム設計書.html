1. システム概要

1.1 プロジェクト概要

本システムは、不動産投資家が物件候補地の立地調査を LINE チャット上で完結できるようにする AIロボットです。ユーザーが住所や地名をメッセージで送信するだけで、Claude AI が国土交通省の地理空間 MCP サーバー（MLIT Geospatial MCP Server）を通じて政府公開データを自動収集・分析し、投資判断に必要な立地レポートを返却します。

1.2 解決する課題

  • 立地調査に必要なデータが複数のWebサイトに散在しており、収集に多大な時間がかかる
  • ハザード情報・都市計画・地価・周辺施設を横断的に確認するのが困難
  • 専門知識がなくても投資リスクを定量的に把握できる手段が少ない
  • LINE という使い慣れたインターフェースで完結できれば操作コストが最小化できる

1.3 システムが提供する価値

価値項目説明
スピード従来数時間かかる立地調査を数分で完了
網羅性価格・ハザード・施設・法規制・人口を一括取得（30種類のAPI）
使いやすさLINEで自然言語のチャット操作のみ。専門知識不要
信頼性国土交通省公式データを直接利用。出典明確
拡張性調査項目や出力形式を柔軟に追加可能

1.4 対象ユーザー

  • 個人不動産投資家（区分マンション・戸建て・土地）
  • 不動産仲介会社のエージェント
  • 不動産ファンド・REITの調査担当者
  • 地方移住・土地活用を検討している一般ユーザー

3. データフローと処理シーケンス

3.1 基本シーケンス

Step処理詳細
Step 1メッセージ受信ユーザーがLINEで「東京都港区六本木の立地調査をして」と送信
Step 2Webhook 転送LINE Messaging API がバックエンドのWebhookエンドポイントにPOST
Step 3処理キューFastAPIがリクエストを受け取り、非同期タスクとしてキューに積む（タイムアウト回避）。ユーザーに「調査中...」メッセージを即時送信
Step 4Claude API 呼び出し会話履歴＋システムプロンプトとともにClaude APIをストリーミング呼び出し
Step 5MCP ツール選択Claude が住所から緯度経度を解析し、必要なMCPツール群を自律的に選択・実行
Step 6API 並列呼び出しMCPサーバーが不動産情報ライブラリの複数APIを並列リクエスト
Step 7データ統合取得した地価・ハザード・施設・法規制・人口データをClaudeが統合分析
Step 8レポート生成構造化されたテキストレポートをClaudeが日本語で生成
Step 9LINE 返信FastAPIがLINE Messaging APIを通じてユーザーに返信（Flex Message形式）

3.2 住所→緯度経度変換（ジオコーディング）

不動産情報ライブラリ APIは緯度経度を入力として受け取るため、住所文字列をまず座標に変換する必要があります。

優先度ジオコーディングサービス特徴
第1候補（推奨）国土地理院 住所検索API無償・国産・申請不要。国内住所に特化
第2候補Google Geocoding API高精度だが有償（月$200 無料枠あり）
第3候補OpenStreetMap Nominatim無償・オープン。利用制限あり（1req/sec以下）

3.3 エラー処理

  • API タイムアウト（30秒）：ユーザーに処理中メッセージを送り、完了後に追送
  • 住所解析失敗：より具体的な住所の入力を案内するメッセージを返却
  • APIレート制限：指数バックオフでリトライ（最大3回）
  • データ未取得：「この地点ではデータが存在しない可能性があります」と明示
  • Claude APIエラー：フォールバックメッセージと再試行の案内

5. LINEインターフェース設計

5.1 基本操作フロー

  1. LINE でBotを「友だち追加」
  2. あいさつメッセージと操作ガイドが表示される
  3. 調査したい住所・地名をテキストで送信（例：「東京都渋谷区恵比寿1-1-1の立地を調査して」）
  4. 「調査中です...少々お待ちください」の中間メッセージが届く（即時返信）
  5. 15〜60秒後に構造化された立地レポートが返信される
  6. 「詳細を見る」「地図を見る」「別の場所を調査する」のクイックリプライが表示される

5.2 コマンド仕様

コマンド動作
（住所/地名のテキスト）その場所の立地調査を実行。自由文でOK（例：「渋谷駅周辺を調べて」）
/help操作方法・利用可能なコマンド一覧を表示
/menu調査メニュー（クイック調査 / 詳細調査 / 比較調査）をボタンで表示
/quick [住所]主要5項目のみ素早く調査（地価・ハザード・最寄り駅・用途地域・将来人口）
/detail [住所]全カテゴリの詳細調査（全30API項目）
/compare [住所A] vs [住所B]2地点の立地を比較分析
/history直近5件の調査履歴を表示
/reset会話をリセット（セッションクリア）

6. AIシステムプロンプト設計

6.1 Claude へのシステムプロンプト

システムプロンプト（draft）あなたは「立地調査ロボット」です。不動産投資家が物件候補地の立地を評価するために、 国土交通省の不動産情報ライブラリAPIを活用して調査を行います。 【役割と行動指針】 1. ユーザーが送信した住所・地名から緯度経度を推定し、必要なAPIツールを自律的に 選択して実行すること。 2. 取得したデータを投資家の視点で分析し、「リスク」と「魅力」を明確に整理すること。 3. 数値データは具体的に引用し、根拠を明示すること。 4. 返答はLINEのメッセージとして読みやすいよう、絵文字と区切り線を効果的に使うこと。 5. 投資判断に直結するポイント（ハザードリスク・地価水準・将来人口）は必ず含めること。 6. 不確実な情報については「データが存在しない可能性があります」と正直に伝えること。 【調査カテゴリ優先順位（ユーザー指定）】 高優先: 価格・地価情報 / ハザード情報 / 周辺施設・利便性 / 都市計画・法規制 / 人口動態 【出力フォーマット】 必ず以下のセクションを含めること： 📍 調査地点の確認 💰 地価・価格情報 ⚠️ ハザード総合評価（リスクレベル：高/中/低） 🏫 生活利便性 🏛️ 法規制・都市計画 👥 人口動態 🤖 AIコメント（総合的な投資判断コメント）

8. セキュリティ・コンプライアンス

8.1 セキュリティ対策

対策項目内容
Webhook署名検証LINE からのリクエストは X-Line-Signature ヘッダーを HMAC-SHA256 で検証
APIキー管理全APIキーはクラウドのSecret Managerで管理。コードへの直書き禁止
HTTPS強制全通信をTLS 1.2以上で暗号化
レート制限ユーザーごとに1分10回・1日100回の上限設定でコスト暴走防止
入力バリデーション住所文字列の長さ・文字種チェック
ログ管理ユーザーIDは匿名化してログ保存。住所テキストは90日で自動削除

8.2 不動産情報ライブラリ利用規約の遵守

9. 事前準備：APIキー取得手順

9.1 不動産情報ライブラリ APIキー（最優先）

  1. https://www.reinfolib.mlit.go.jp/api/request/ にアクセス
  2. 利用者種別（法人 / 法人以外の団体 / 個人）を選択
  3. 担当者氏名・メールアドレスを入力して申請送信
  4. 審査後（数日〜1週間程度）にメールでAPIキーが送付される
  5. 取得したキーを LIBRARY_API_KEY として環境変数に設定

9.2 LINE Messaging API

  1. LINE Developers（https://developers.line.biz/）にログイン
  2. プロバイダーを作成（または既存のものを選択）
  3. 新規チャンネル → Messaging API を選択
  4. チャンネル設定から「チャンネルシークレット」と「チャンネルアクセストークン」を取得
  5. Webhook URLにデプロイ済みエンドポイントを設定（例：https://your-domain.com/webhook）
  6. 「Webhookの利用」をオンにし、「接続確認」でOKを確認

9.3 Claude API（Anthropic）

  1. https://console.anthropic.com/ にアクセス
  2. アカウントを作成またはログイン
  3. 「API Keys」セクションから新規キーを発行
  4. 取得したキーを ANTHROPIC_API_KEY として環境変数に設定
  5. 利用モデル：claude-sonnet-4-6（推奨。コスト効率と性能のバランスが良い）

9.4 ジオコーディングAPI（住所→座標変換）

サービス推奨度特徴・取得先
