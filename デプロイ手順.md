# デプロイ手順（GitHub → Render）

## 前提
- [x] GitHubアカウントあり
- [x] Renderアカウントあり
- [x] 4つのAPIキーすべて取得済み

---

## ステップ1：GitHubにリポジトリを作成してpush

1. GitHub で新しいリポジトリを作成（例：`locagenius`）
2. このフォルダ内のファイルをpush

```bash
cd /Users/tomita/Documents/Claude/Projects/LocaGenius
git init
git add .
git commit -m "initial commit"
git remote add origin https://github.com/あなたのユーザー名/locagenius.git
git push -u origin main
```

> ⚠️ `.env` ファイルは `.gitignore` で除外されているのでpushされません。

---

## ステップ2：RenderにWebサービスを作成

1. [render.com](https://render.com) にログイン
2. 「New +」→「Web Service」を選択
3. 「Build and deploy from a Git repository」→ GitHubと連携して `locagenius` リポジトリを選択
4. 以下の設定を確認：
   - **Environment**: Docker（`render.yaml` が自動で読み込まれる）
   - **Plan**: Starter（$7/月）
     - ※ Free プランはスリープがあり、LINEのWebhookが失敗することがある

---

## ステップ3：環境変数を設定

Renderダッシュボードの「Environment」タブで以下を入力：

| キー | 値 |
|------|-----|
| `LINE_CHANNEL_SECRET` | LINEのチャンネルシークレット |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINEのチャンネルアクセストークン |
| `ANTHROPIC_API_KEY` | AnthropicのAPIキー |
| `LIBRARY_API_KEY` | 不動産情報ライブラリのAPIキー |

> `MCP_SERVER_PATH` は `render.yaml` に記載済みなので不要。

---

## ステップ4：デプロイ完了を確認

1. 「Deploy」ボタンでデプロイ開始（初回は5〜10分かかる）
2. ログに `Application startup complete` が出ればOK
3. デプロイされたURL（例：`https://locagenius.onrender.com`）を確認

---

## ステップ5：LINE Webhook URLを設定

1. [LINE Developers](https://developers.line.biz/) にログイン
2. Messaging API チャンネルを開く
3. 「Messaging API設定」→「Webhook URL」に以下を入力：
   ```
   https://あなたのURL.onrender.com/webhook
   ```
4. 「検証」ボタンをクリックして `200 OK` が返ることを確認
5. 「Webhookの利用」を **オン** に設定

---

## ステップ6：動作テスト

1. LINE でボットを友だち追加
2. 「こんにちは」と送ってあいさつメッセージが返ってくることを確認
3. 「東京都渋谷区恵比寿1-1-1」と送って立地調査が動くことを確認
   - 「調査を開始します...」がすぐ返ってくる
   - 30秒〜1分後に調査結果が届く

---

## トラブルシューティング

| 症状 | 原因と対処 |
|------|-----------|
| Webhookの検証が失敗する | Renderのデプロイが完了していない。ログを確認 |
| 「Invalid signature」エラー | `LINE_CHANNEL_SECRET` が間違っている |
| 調査結果が返ってこない | RenderのログでPythonエラーを確認。APIキーの設定ミスが多い |
| 調査が遅い（2分以上） | MCPサーバーの起動に時間がかかっている。正常範囲は30〜60秒 |
